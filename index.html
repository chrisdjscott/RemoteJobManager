Welcome to RemoteJobManager's documentation!¶

RemoteJobManager (RJM) provides an API to offload work to a remote system. It uses Globus to transfer files and funcX to execute commands on the remote machine, but could be extended to use other similar mechanisms (e.g. Slurm API). It has been developed for use on NeSI's HPC platform (Slurm cluster with Globus support).

Some "one-time setup" and installation and configuration of RJM are required before using the tool.
